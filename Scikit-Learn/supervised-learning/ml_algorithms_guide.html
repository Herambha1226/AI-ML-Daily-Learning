01

Linear Regression

Regression
What it does

Fits a straight line (or hyperplane) through data to predict a continuous numeric output from one or more input features.

Where to use

House price prediction, sales forecasting, temperature estimation, any problem where output is a real number and relationships are roughly linear.

Why it works

Minimizes the sum of squared differences between predicted and actual values — a convex problem with a closed-form solution.

Core Formulas

Hypothesis / Prediction
ŷ = θ₀ + θ₁x₁ + θ₂x₂ + ... + θₙxₙ = Xθ
ŷ = predicted value  |  θ₀ = bias (intercept)  |  θ₁…θₙ = weights (coefficients)  |  X = feature matrix
Cost Function — Mean Squared Error (MSE)
J(θ) = (1/2m) Σᵢ (ŷᵢ − yᵢ)²
m = number of training samples  |  yᵢ = actual value  |  The ½ simplifies the gradient. We want to minimize J(θ).
Closed-Form Solution (Normal Equation)
θ = (XᵀX)⁻¹ Xᵀy
Directly computes optimal weights without iteration. Works well for small datasets (n < 10,000 features). Requires matrix inversion O(n³).
Gradient Descent Update Rule
θⱼ := θⱼ − α · (∂J/∂θⱼ) = θⱼ − (α/m) Σᵢ (ŷᵢ−yᵢ)xᵢⱼ
α = learning rate (step size)  |  Partial derivative tells us the slope of the cost w.r.t. each weight. Applied simultaneously for all θⱼ.

Key Mathematics Used

∑

Linear Algebra

Matrix multiplication (Xθ), transpose (Xᵀ), matrix inversion — core to the Normal Equation and vectorized computation.

∂

Partial Derivatives / Calculus

∂J/∂θⱼ computes the gradient of the loss w.r.t. each weight. Setting it to zero gives the optimal solution analytically.

‖·‖²

L2 Norm / Euclidean Distance

MSE loss is proportional to the squared L2 norm of the residual vector (y − Xθ).

R²

Statistics — R² Score

R² = 1 − SS_res/SS_tot. Measures proportion of variance explained. R²=1 is perfect, R²=0 means model predicts the mean.

Algorithm Steps

Initialize Weights

Set θ = [0, 0, …, 0] or small random values. Also initialize bias θ₀ = 0.

Compute Predictions

For all samples: ŷ = Xθ (matrix multiplication gives predictions in one step).

ŷ = Xθ

Compute Loss (MSE)

Calculate how wrong predictions are: J(θ) = (1/2m)||ŷ − y||²

J = (1/2m) Σ(ŷ−y)²

Compute Gradient

∂J/∂θ = (1/m) Xᵀ(Xθ − y). This is the direction of steepest ascent.

∇J = (1/m) Xᵀ(ŷ − y)

Update Weights

Move in the opposite direction of gradient: θ := θ − α∇J. Repeat until convergence.

θ := θ − α · ∇J

Check Convergence

Stop when ||J(θ_new) − J(θ_old)|| < ε (tolerance), or after max iterations. Return final θ.

Polynomial Regression

Regression
What it does

Extends linear regression by adding polynomial terms (x², x³ …) as new features, enabling it to fit curved, non-linear data.

Where to use

Growth curves, dose-response relationships, physics simulations, any domain where relationships are curved but smooth.

Why it works

By creating new features (x², x³ …) the problem becomes linear in the new feature space — so linear regression math still applies.

Core Formulas

Degree-d Polynomial Hypothesis
ŷ = θ₀ + θ₁x + θ₂x² + θ₃x³ + ... + θ_d x^d
For multiple features x₁, x₂: also includes cross-terms like x₁x₂, x₁²x₂, etc. Feature engineering creates these new columns.
Feature Mapping (Transformation)
φ(x) = [1, x, x², x³, ..., x^d]
The original feature x is mapped to a higher-dimensional vector φ(x). The model is then linear: ŷ = θᵀ φ(x).
Bias-Variance Tradeoff
Total Error = Bias² + Variance + Irreducible Noise
Low degree (d=1) → high bias, low variance (underfitting). High degree (d=10) → low bias, high variance (overfitting). Choose d by cross-validation.

Algorithm Steps

Choose Polynomial Degree d

Use domain knowledge or cross-validation to select d. Start with d=2 or d=3.

Transform Features

Create new feature matrix: [x, x², x³, …, x^d]. For multi-feature: also include interaction terms.

X_poly = PolynomialFeatures(d).fit_transform(X)

Apply Linear Regression

Run standard linear regression on the transformed feature matrix. All linear regression math now applies.

Regularize (if needed)

High-degree polynomials overfit easily. Add L1 (Lasso) or L2 (Ridge) regularization to the weights.

03

Logistic Regression

Classification
What it does

Predicts the probability that an input belongs to a class (0 or 1). Uses the sigmoid function to squash output between 0 and 1.

Where to use

Email spam detection, disease diagnosis (yes/no), credit risk scoring, any binary or multi-class classification with probabilistic output.

Why it works

Models the log-odds as a linear function, then converts to probability via sigmoid. Maximum likelihood estimation finds optimal weights.

Core Formulas

Sigmoid (Logistic) Function
σ(z) = 1 / (1 + e^(−z))    where z = θᵀx
Output is always in (0,1). Interpreted as P(y=1|x). If σ(z) ≥ 0.5 → predict class 1; else → class 0.
Log Loss (Binary Cross-Entropy)
J(θ) = −(1/m) Σ [y·log(ŷ) + (1−y)·log(1−ŷ)]
Penalizes confident wrong predictions heavily. Convex function — guaranteed to find global minimum with gradient descent.
Gradient (Gradient Descent Update)
∂J/∂θ = (1/m) Xᵀ(σ(Xθ) − y)
Identical in form to linear regression gradient — the sigmoid is built into ŷ = σ(Xθ). Update: θ := θ − α · ∂J/∂θ
Log-Odds (Logit Function)
log(p / (1−p)) = θ₀ + θ₁x₁ + ... + θₙxₙ
The left side is the log-odds (logit). Logistic regression models this as linear in x. Exponentiating gives the odds ratio.

Key Mathematics

e^z

Exponential / Sigmoid

The natural exponential e^z compresses any real-valued linear combination into probability (0,1).

log

Log Likelihood

Maximum Likelihood Estimation (MLE) maximizes log P(y|X,θ). Negating gives the cross-entropy loss to minimize.

∂²

Convexity

Log loss is strictly convex — Hessian matrix is positive semi-definite — so gradient descent always finds global minimum.

🎯

Decision Boundary

θᵀx = 0 is the decision boundary. Points on either side get different class predictions. Can be non-linear with polynomial features.

Algorithm Steps

Initialize Weights θ

Set all weights to 0 or small random values. θ ∈ ℝⁿ⁺¹ (n features + bias).

Compute Linear Combination z

z = Xθ (matrix multiply input features with weight vector).

z = θ₀ + θ₁x₁ + ... + θₙxₙ

Apply Sigmoid → Get Probabilities

ŷ = σ(z) = 1/(1+e^(−z)). Each output ŷᵢ ∈ (0,1) is P(y=1 | xᵢ).

ŷ = 1 / (1 + exp(−Xθ))

Compute Cross-Entropy Loss

J = −(1/m) Σ [y·log(ŷ) + (1−y)·log(1−ŷ)]

Compute Gradient & Update

∇J = (1/m) Xᵀ(ŷ−y). Update θ := θ − α·∇J. Repeat until convergence.

Predict Class Labels

ŷ_class = 1 if σ(θᵀx) ≥ 0.5 else 0. Threshold can be adjusted for precision/recall tradeoff.

04

Support Vector Machines

Classification Regression (SVR)
What it does

Finds the maximum-margin hyperplane that best separates two classes. Only the support vectors (closest points) define the boundary.

Where to use

Text classification, image recognition, bioinformatics (protein classification), small-to-medium high-dimensional datasets.

Why it works

Maximum margin = best generalization. The kernel trick maps data to higher dimensions implicitly without computing the transformation.

Core Formulas

Decision Function / Hyperplane
f(x) = wᵀx + b    Predict: sign(wᵀx + b)
w = weight vector (normal to hyperplane)  |  b = bias  |  Margin = 2 / ||w||. Goal: maximize margin.
Hard Margin SVM (Primal Objective)
minimize: ½||w||²    subject to: yᵢ(wᵀxᵢ + b) ≥ 1 ∀i
Convex quadratic programming problem. yᵢ ∈ {−1, +1}. The constraint ensures all points are correctly classified with margin ≥ 1.
Soft Margin SVM (with Slack Variables)
minimize: ½||w||² + C Σᵢ ξᵢ    subject to: yᵢ(wᵀxᵢ+b) ≥ 1−ξᵢ, ξᵢ ≥ 0
ξᵢ = slack variable (how much point i violates margin)  |  C = regularization: large C = less tolerance for violations (risk overfitting).
Kernel Trick
K(xᵢ, xⱼ) = φ(xᵢ)ᵀφ(xⱼ)
RBF Kernel: K(x,z) = exp(−γ||x−z||²)  |  Polynomial: K(x,z) = (xᵀz + c)^d  |  Avoids explicit computation of φ(x).
Dual Problem (Lagrangian)
maximize: Σαᵢ − ½ Σᵢ Σⱼ αᵢαⱼyᵢyⱼ K(xᵢ,xⱼ)
αᵢ = Lagrange multipliers. Non-zero αᵢ → support vectors. Decision: f(x) = Σᵢ αᵢ yᵢ K(xᵢ,x) + b

Algorithm Steps

Choose Kernel & Hyperparameters

Select kernel (linear, RBF, poly), set C (regularization), γ (RBF bandwidth). Use cross-validation / grid search.

Compute Kernel Matrix K

K[i,j] = K(xᵢ, xⱼ) — an m×m matrix of pairwise similarities. This is the Gram matrix.

K[i,j] = exp(−γ ||xᵢ − xⱼ||²)

Solve Quadratic Programming Problem

Find Lagrange multipliers αᵢ ≥ 0 that maximize the dual objective using QP solvers (SMO algorithm in practice).

Identify Support Vectors

Points with αᵢ > 0 are support vectors. Only these matter for prediction — others are discarded.

Compute Bias b

For any support vector: b = yᵢ − Σⱼ αⱼ yⱼ K(xⱼ, xᵢ). Average over all support vectors for stability.

Predict New Points

f(x) = sign(Σᵢ αᵢ yᵢ K(xᵢ, x) + b). Sum is only over support vectors (αᵢ > 0).

05

K-Nearest Neighbors

Classification & Regression
What it does

For each test point, finds the k most similar training points (neighbors) and predicts by majority vote (classification) or average (regression).

Where to use

Recommendation systems, anomaly detection, imputing missing values, small-to-medium datasets where interpretability matters.

Why it works

Assumes nearby points have similar labels (locality assumption). No training phase — the entire dataset IS the model (lazy learning).

Core Formulas

Euclidean Distance (most common)
d(x, xᵢ) = √(Σⱼ (xⱼ − xᵢⱼ)²) = ||x − xᵢ||₂
Straight-line distance in feature space. Sensitive to scale — always normalize features first (z-score or min-max scaling).
Manhattan Distance
d(x, xᵢ) = Σⱼ |xⱼ − xᵢⱼ| = ||x − xᵢ||₁
Sum of absolute differences. Better for high-dimensional spaces. Less sensitive to outliers than Euclidean.
Minkowski Distance (General)
d(x, xᵢ) = (Σⱼ |xⱼ − xᵢⱼ|^p)^(1/p)
p=1 → Manhattan, p=2 → Euclidean, p=∞ → Chebyshev (max coordinate diff). Choose p via cross-validation.
KNN Prediction
Classification: ŷ = argmax_c Σᵢ∈N_k 𝟙[yᵢ=c]    Regression: ŷ = (1/k) Σᵢ∈N_k yᵢ
N_k = set of k nearest neighbors  |  Classification = majority vote  |  Regression = arithmetic mean of neighbor labels.
Weighted KNN
ŷ = Σᵢ∈N_k wᵢ·yᵢ / Σᵢ∈N_k wᵢ    where wᵢ = 1/d(x, xᵢ)
Closer neighbors vote more. Inverse distance weighting reduces the arbitrary nature of hard k cutoff. Better in practice.

Algorithm Steps

Choose k & Distance Metric

k = number of neighbors. Odd k avoids ties. Use cross-validation. More k → smoother boundary but may miss patterns.

Normalize Features

Scale all features to same range (z-score: x' = (x−μ)/σ or min-max: x' = (x−min)/(max−min)). Critical for distance metrics.

x' = (x − μ) / σ

Store Training Data

No model training! Simply store all (xᵢ, yᵢ) pairs. Use KD-Tree or Ball-Tree for efficient lookup in high dimensions.

Compute Distances to All Training Points

For query x_new: compute d(x_new, xᵢ) for all i. O(n·d) per prediction where n = training size, d = dimensions.

Find k Nearest Neighbors

Sort distances and select k smallest. These are the neighbors N_k = {x(1), x(2), …, x(k)}.

Aggregate Labels

Classification: majority vote. Regression: mean (or weighted mean). Return prediction.

ŷ = majority_vote(y₁, y₂, ..., yₖ)

Decision Trees

Classification & Regression
What it does

Recursively splits data on the feature threshold that best separates the classes. Creates an if-then tree of decisions.

Where to use

Medical diagnosis, customer segmentation, fraud rules, any problem needing interpretable, human-readable decision rules.

Why it works

Greedily maximizes information gain (or Gini reduction) at each split, partitioning the feature space into axis-aligned rectangles.

Core Formulas

Gini Impurity (CART)
Gini(S) = 1 − Σₖ pₖ² = 1 − Σₖ (nₖ/n)²
pₖ = fraction of class k in node S. Gini = 0 → pure node (all same class). Gini = 0.5 → maximally impure (equal classes). Used in scikit-learn by default.
Entropy (Information Gain)
H(S) = −Σₖ pₖ log₂(pₖ)
Shannon entropy measures disorder. Pure node → H = 0. Uniform → H = log₂(K). Information Gain: IG = H(parent) − Σ (|child|/|parent|)·H(child)
Best Split Selection
split* = argmax_(feature j, threshold t) [ H(S) − Σ (|Sᵥ|/|S|)·H(Sᵥ) ]
For each feature j and each threshold t: compute weighted impurity of left/right children. Choose (j, t) that maximizes information gain.
Variance Reduction (Regression Trees)
Gain = Var(S) − Σᵥ (|Sᵥ|/|S|)·Var(Sᵥ)
For regression: minimize MSE within each child. Predict the mean of training values in the leaf: ŷ_leaf = (1/|leaf|) Σ yᵢ

Algorithm Steps (CART / ID3)

Start at Root Node with All Data

The root contains all m training samples. Compute current impurity H(root).

Find Best Split

For every feature j and every threshold t: split data into left (xⱼ < t) and right (xⱼ ≥ t). Compute information gain. Pick (j*, t*) with max gain.

IG(j,t) = H(parent) − wₗH(left) − wᵣH(right)

Split the Node

Create left and right child nodes. Assign samples according to x[j*] < t* (left) or ≥ t* (right).

Recurse on Children

Apply steps 2–3 recursively on each child. Stop when: node is pure, max_depth reached, min_samples_split not met, or no gain.

Assign Leaf Predictions

Classification: most frequent class. Regression: mean of y values in leaf.

Prune (Optional)

Post-prune: remove branches that don't improve validation accuracy. Prevents overfitting. Or use max_depth, min_samples_leaf as pre-pruning.

07

Random Forest

Ensemble Classification & Regression
What it does

Trains many decision trees independently on bootstrapped subsets of data with random feature subsets, then aggregates predictions.

Where to use

Tabular data competitions, medical imaging, financial modeling, feature selection, any task where a single tree overfits.

Why it works

Bagging + random feature selection decorrelates trees. Averaging uncorrelated noisy predictors reduces variance without increasing bias.

Core Formulas

Bootstrap Sampling
For tree t: Sample m points WITH replacement from training set D → Dₜ
~63% of unique samples appear in each bootstrap bag. The remaining ~37% are "out-of-bag" (OOB) samples, useful for free validation.
Random Feature Subsampling
At each split: select √p features randomly (classification) or p/3 (regression)
p = total number of features. Random subsampling ensures trees are different even on similar bootstrap samples — decorrelation key.
Ensemble Prediction
Classification: ŷ = majority_vote{T₁(x), T₂(x), ..., Tₙ(x)}
Regression: ŷ = (1/B) Σₜ Tₜ(x)    B = number of trees. More trees → lower variance, diminishing returns after ~100–500 trees.
Bias-Variance Decomposition
Var(avg of B trees) = ρ·σ² + (1−ρ)σ²/B
ρ = average correlation between trees  |  σ² = variance of single tree. As B→∞, variance → ρσ². Lower ρ (via random features) → lower final variance.
Feature Importance (Mean Decrease Impurity)
FI(j) = Σₜ Σₙ (|Sₙ|/|S|) · ΔImpurity(n, feature j)
Sum over all trees t and all nodes n where feature j was used: weighted impurity decrease. Normalized to sum to 1.

Algorithm Steps

Set Hyperparameters

n_estimators (number of trees), max_depth, min_samples_split, max_features (√p or p/3), random_state.

Bootstrap Sample for Each Tree

Draw m samples with replacement from training data. Each tree sees a different ~63% of data.

Dₜ = bootstrap_sample(D, size=m)

Grow Deep Tree with Random Features

At each node split: randomly select √p features, then find the best split among ONLY those features. Grow to max depth (no pruning).

Repeat for B Trees

All trees are trained independently (parallelizable). Store all B trees.

Aggregate Predictions

For new x: get prediction from each tree, then majority vote (classification) or mean (regression).

ŷ = (1/B) Σ Tₜ(x)

Compute OOB Score (Free Validation)

For each training point, average predictions from trees that did NOT see it (OOB samples). Unbiased estimate of generalization error.

08

Gradient Boosting Machines

Ensemble Classification & Regression
What it does

Builds trees sequentially, each one correcting the errors (residuals) of the previous. Gradient descent in function space.

Where to use

Kaggle competitions, click-through-rate prediction, ranking problems, structured/tabular data. XGBoost, LightGBM, CatBoost are variants.

Why it works

Fits each new tree to the negative gradient of the loss — this is gradient descent on the space of functions, not parameters.

Core Formulas

Ensemble Model (Additive)
F_M(x) = F₀(x) + Σₘ₌₁ᴹ η·hₘ(x)
F₀ = initial model (e.g., mean of y)  |  hₘ = m-th tree  |  η = learning rate (shrinkage). Trees are added sequentially.
Pseudo-Residuals (Negative Gradient)
rᵢₘ = −[∂L(yᵢ, F(xᵢ)) / ∂F(xᵢ)]_{F=Fₘ₋₁}
For MSE: rᵢ = yᵢ − ŷᵢ (actual residuals). For log-loss: rᵢ = yᵢ − σ(ŷᵢ). The new tree hₘ is fit to predict these residuals.
Line Search / Leaf Values
γⱼₘ = argmin_γ Σᵢ∈Rⱼₘ L(yᵢ, Fₘ₋₁(xᵢ) + γ)
For each leaf region Rⱼₘ: find optimal output value γ that minimizes loss. For MSE: γ = mean of residuals in leaf.
XGBoost Objective (2nd-Order Approximation)
Obj ≈ Σᵢ [gᵢ·fₜ(xᵢ) + ½hᵢ·fₜ(xᵢ)²] + Ω(fₜ)
gᵢ = first-order gradient  |  hᵢ = second-order gradient (Hessian)  |  Ω(f) = regularization term on tree complexity. Enables faster, better optimization.

Algorithm Steps

Initialize with Constant Prediction

F₀(x) = argmin_γ Σᵢ L(yᵢ, γ). For MSE: F₀ = mean(y). For log-loss: F₀ = log(mean_y / (1−mean_y)).

F₀ = mean(y) or log(p̄/(1−p̄))

Compute Pseudo-Residuals

rᵢ = −∂L(yᵢ, Fₘ₋₁(xᵢ))/∂F. These are the "errors" the next tree should correct.

rᵢ = yᵢ − Fₘ₋₁(xᵢ) (for MSE)

Fit a Shallow Tree hₘ to Residuals

Regress the pseudo-residuals using a decision tree (typically max_depth 3–8). Tree learns which features explain the errors.

Compute Leaf Output Values

Optimal leaf value = average of residuals in leaf (MSE). More generally: solve argmin γ over L for each leaf.

Update the Ensemble

Fₘ(x) = Fₘ₋₁(x) + η·hₘ(x). Learning rate η (0.01–0.3) shrinks contribution of each tree → better generalization.

Fₘ = Fₘ₋₁ + η · hₘ

Repeat for M iterations

Steps 2–5 repeat M times (n_estimators). Use early stopping: halt if validation loss doesn't improve for k rounds.

Lasso & Ridge Regression

Regularization Regression
What it does

Adds a penalty term to the linear regression cost function to shrink weights and prevent overfitting. Lasso (L1) can eliminate features; Ridge (L2) shrinks them.

Where to use

High-dimensional data (many features), multicollinearity, feature selection (Lasso), when all features are relevant (Ridge), genomics, text, finance.

Why it works

Regularization constrains the hypothesis space. L1 creates sparse solutions (many θ = 0) due to the diamond-shaped constraint. L2 distributes weight evenly.

Core Formulas

Ridge Regression (L2 Regularization)
J(θ) = (1/2m) Σ(ŷᵢ−yᵢ)² + λ·Σⱼ θⱼ²
λ (alpha) = regularization strength  |  Penalty = sum of squared weights. Closed form: θ = (XᵀX + λI)⁻¹ Xᵀy. Always invertible even if XᵀX is singular.
Lasso Regression (L1 Regularization)
J(θ) = (1/2m) Σ(ŷᵢ−yᵢ)² + λ·Σⱼ |θⱼ|
Penalty = sum of absolute weights. Non-differentiable at θ=0 → requires coordinate descent or subgradient methods. Produces sparse θ (feature selection).
Ridge Gradient (Closed Form)
θ_ridge = (XᵀX + λI)⁻¹ Xᵀy
The λI term ensures the matrix is always invertible. Gradient: ∂J/∂θ = (1/m) Xᵀ(Xθ−y) + 2λθ. Update: θⱼ := θⱼ(1−2αλ) − (α/m)Σᵢ(ŷᵢ−yᵢ)xᵢⱼ
Lasso Soft-Thresholding (Coordinate Descent)
θⱼ = sign(ρⱼ) · max(|ρⱼ| − λ, 0)
ρⱼ = partial residual correlation with feature j. If |ρⱼ| < λ → θⱼ = 0 (feature eliminated!). This is the soft-thresholding operator — key to sparsity.

Lasso vs Ridge: Key Differences

Constraint Shape

Ridge: circular/spherical constraint ||θ||₂ ≤ t. Lasso: diamond/hypercube ||θ||₁ ≤ t. The corners of the L1 diamond are on axes → sparse solutions.

Effect on Coefficients

Ridge: shrinks all θⱼ toward 0 but never exactly 0. Lasso: can set θⱼ = exactly 0 → automatic feature selection. Use Lasso when many features are irrelevant.

Multicollinearity

Ridge is better when features are correlated (distributes weight evenly). Lasso arbitrarily picks one correlated feature and drops others.

Choosing λ

Use k-fold cross-validation. Plot validation error vs log(λ). Too large → underfitting; too small → overfitting. LassoCV / RidgeCV automate this.

λ* = argmin CV_error(λ)
10

ElasticNet Regularization

Regularization
What it does

Combines L1 (Lasso) and L2 (Ridge) penalties linearly. Gets the best of both: sparsity from L1 and grouping from L2.

Where to use

Genomics (p >> n), text mining, when there are correlated feature groups and you want sparse solutions. Better than pure Lasso for grouped features.

Why it works

L1 encourages sparsity, L2 handles correlated features together. l1_ratio controls the mix — when correlated features exist, all are selected or dropped together.

Core Formulas

ElasticNet Cost Function
J(θ) = (1/2m) Σ(ŷᵢ−yᵢ)² + λ·[ρ·Σ|θⱼ| + (1−ρ)/2 · Σθⱼ²]
λ = overall regularization strength  |  ρ (l1_ratio) = mix between L1 and L2. ρ=1 → pure Lasso; ρ=0 → pure Ridge; ρ=0.5 → equal mix.
Coordinate Descent Update
θⱼ = sign(ρⱼ) · max(|ρⱼ| − λρ, 0) / (1 + λ(1−ρ))
Numerator: soft-thresholding (L1 part). Denominator: 1 + λ(1−ρ) is the L2 shrinkage. Both happen simultaneously in each coordinate update step.
Naive Elastic Net
Augment X with √λ₂·I and augment y with 0. Then apply Lasso.
The classic trick: adding p artificial features allows Lasso coordinate descent to solve the ElasticNet. Same algorithm, larger system.

Algorithm Steps

Set λ and ρ (l1_ratio)

Tune both via cross-validation. Common: ElasticNetCV with a grid of l1_ratios [0.1, 0.5, 0.7, 0.9, 0.95, 1.0].

Initialize Weights θ = 0

Start from the zero vector. Coordinate descent converges from any starting point for ElasticNet (convex).

Coordinate Descent Loop

For each feature j: compute partial residual ρⱼ = Xⱼᵀ(y − X_{-j}θ_{-j}). Apply combined soft-threshold update for θⱼ.

θⱼ ← S(ρⱼ, λρ) / (1 + λ(1−ρ))

Cycle Until Convergence

Repeat coordinate updates for all j in order until max change in θ < tolerance. Typically converges in < 1000 passes.

Key Concepts Glossary

WHAT · WHY · WHERE · HOW — Every term explained

Bias (θ₀ / b / intercept)
ŷ = θ₀ + θ₁x₁ + ...

WHY Without bias, the model is forced through the origin — limits flexibility. Bias shifts the decision boundary or regression line.

WHERE Every parametric model: linear regression, logistic regression, neural networks, SVMs.

HOW Treat it as an extra weight with constant input x₀=1. Updated by gradient descent like any other weight.

Weights / Coefficients (θ, w)
ŷ = Σⱼ θⱼ·xⱼ

WHY Each weight determines how much each feature influences the output. Learned from data by minimizing loss.

WHERE All linear models, neural networks, SVMs. In trees: not explicit — splitting thresholds play the role of weights.

HOW Updated via gradient descent: θⱼ := θⱼ − α·∂J/∂θⱼ. Represents the marginal effect of feature j on prediction.

Gradient Descent
θ := θ − α·∇J(θ)

WHY Most loss functions don't have closed-form solutions. Gradient descent iteratively follows the steepest downhill direction.

WHERE Linear regression, logistic regression, neural networks, gradient boosting.

HOW Batch GD: uses all data; SGD: one sample; Mini-batch: k samples. Learning rate α controls step size.

Learning Rate (α)
θ := θ − α · ∇J

WHY Too small → slow convergence. Too large → overshoots, may diverge. α is the most critical hyperparameter in gradient-based methods.

WHERE All gradient-based optimization: linear regression, logistic regression, GBM, neural nets.

HOW Tune via learning rate schedulers, warm restarts, or adaptive methods (Adam, RMSprop). Typical range: 0.001–0.1.

Loss / Cost Function
MSE: Σ(ŷ−y)²/m, Cross-Entropy: −Σy·log(ŷ)

WHY Quantifies "how wrong" the model is. Minimizing loss = making better predictions. The choice of loss determines what the model optimizes.

WHERE MSE for regression; Cross-entropy for classification; Hinge loss for SVM.

HOW Computed over training set, differentiated w.r.t. weights to get gradient direction for update.

Overfitting & Underfitting
Error = Bias² + Variance + Noise

WHY Overfitting: model memorizes training data, fails on new data (high variance). Underfitting: model too simple, misses patterns (high bias).

WHERE All ML models. High-degree polynomial and deep trees overfit; linear models may underfit.

HOW Detect via train vs. validation loss gap. Fix overfitting: regularization, more data, dropout. Fix underfitting: more complex model, more features.

Regularization (λ / C)
J_reg = J_data + λ · penalty(θ)

WHY Constrains weight magnitudes to prevent overfitting. Adds a penalty for complexity to the loss function.

WHERE Ridge (L2), Lasso (L1), ElasticNet. Also: dropout (neural nets), max_depth (trees), C in SVM.

HOW L1: |θ| penalty → sparsity. L2: θ² penalty → small weights. Tune λ via cross-validation. Larger λ = more regularization.

Kernel (SVM)
K(x,z) = φ(x)ᵀφ(z)

WHY Maps data to higher-dimensional space where it becomes linearly separable. The "kernel trick" computes K(x,z) without ever computing φ explicitly.

WHERE SVM, Gaussian Processes, Kernel PCA, kernel regression.

HOW Linear kernel for high-dim text; RBF for smooth boundaries; Polynomial for explicit degree-d boundaries.

Entropy & Information Gain
H = −Σ p·log₂(p), IG = H(parent) − Σwᵢ·H(child)

WHY Measures impurity/disorder in a node. Higher information gain → the split is more useful for separating classes.

WHERE Decision trees (ID3, C4.5). Gini impurity is the alternative used by CART/scikit-learn.

HOW Compute entropy before and after each possible split. Choose the split with maximum information gain or minimum Gini.

Support Vectors
αᵢ > 0 ↔ xᵢ is a support vector

WHY Only the training points closest to the decision boundary (support vectors) define the margin and influence the model. Others are irrelevant.

WHERE SVM classification and SVR regression.

HOW Identified via Lagrange multipliers αᵢ from the dual problem. Non-zero αᵢ → support vector. Prediction: f(x) = Σ αᵢyᵢK(xᵢ,x) + b.

Bagging (Bootstrap Aggregating)
ŷ = (1/B) Σᵦ Tᵦ(x)

WHY Reduces variance by averaging multiple models trained on different bootstrap samples. Works best with high-variance base learners (deep trees).

WHERE Random Forest (bagging + random features). Also standalone BaggingClassifier.

HOW Sample m points with replacement, train a model, repeat B times. Average (or majority vote) predictions from all B models.

Boosting
Fₘ = Fₘ₋₁ + η·hₘ

WHY Reduces bias by sequentially correcting errors. Each new model focuses on previously misclassified or poorly predicted points.

WHERE GBM, XGBoost, LightGBM, CatBoost, AdaBoost. State-of-the-art on tabular data.

HOW Fit model to pseudo-residuals (negative gradient of loss). Shrink its contribution by η. Repeat M times.

Cross-Entropy Loss
L = −[y·log(p) + (1−y)·log(1−p)]

WHY Heavily penalizes confident wrong predictions (log(0) → ∞). Derived from MLE under Bernoulli distribution. Convex — guaranteed global minimum.

WHERE Logistic regression, neural net classification, gradient boosting for classification.

HOW Binary: −[y·log(ŷ) + (1−y)·log(1−ŷ)]. Multi-class: −Σₖ yₖ·log(ŷₖ) (categorical cross-entropy).

Feature Scaling / Normalization
z-score: x' = (x−μ)/σ, min-max: x' = (x−min)/(max−min)

WHY Unscaled features cause gradient descent to take inefficient zigzag paths. Distance metrics (KNN, SVM) are completely wrong without scaling.

WHERE KNN, SVM, Logistic Regression, Linear Regression, Neural Nets. Not needed: Decision Trees, Random Forest, GBM (tree splits are scale-invariant).

HOW StandardScaler (z-score) for normally distributed features; MinMaxScaler for bounded ranges; RobustScaler for outliers.

Curse of Dimensionality
Vol(unit ball) → 0 as d → ∞

WHY In high dimensions, all points become equidistant. KNN breaks down because "nearest neighbor" is not meaningfully close. More features ≠ better.

WHERE KNN (severely affected), SVM (moderately), clustering. Trees are relatively robust.

HOW Mitigate with dimensionality reduction (PCA, t-SNE), feature selection, regularization, domain knowledge.

MLE (Maximum Likelihood Estimation)
θ* = argmax_θ Σᵢ log P(yᵢ | xᵢ, θ)

WHY Principled statistical foundation: find parameters that make the observed data most probable under the assumed distribution.

WHERE Logistic regression (MLE ↔ cross-entropy), linear regression (MLE under Gaussian noise ↔ MSE), GBMs.

HOW Log-likelihood is concave → gradient ascent finds global max. Negating gives cross-entropy loss for gradient descent.

Hyperparameter vs Parameter
θ = learned, H = set before training

WHY Parameters (θ, w) are learned from data. Hyperparameters (k in KNN, λ in Ridge, max_depth in trees) control the learning process itself — not learned.

WHERE Every ML algorithm has both. Grid search, random search, Bayesian optimization tune hyperparameters.

HOW Use k-fold cross-validation to estimate generalization performance for each hyperparameter combination.

Gini Impurity
Gini(S) = 1 − Σₖ pₖ²

WHY Measures how often a randomly chosen element would be misclassified. Pure node (one class) → Gini=0. Used in CART and scikit-learn's Decision Tree.

WHERE Decision Trees, Random Forests. Also used to compute feature importance scores.

HOW At each node split, compute weighted Gini of children: Gain = Gini(parent) − Σᵥ (|Sᵥ|/|S|)·Gini(Sᵥ). Pick split with max gain.
